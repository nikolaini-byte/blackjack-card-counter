# Mitwirken & Community

## 🌟 Erste Schritte

Wir freuen uns über Ihr Interesse am Blackjack Card Counter Projekt! Egal, ob Sie Entwickler, Designer, Tester oder einfach nur begeisterter Blackjack-Spieler sind – Ihre Beiträge sind willkommen.

### Schnellstart für Entwickler

1. **Repository forken**
   Klicken Sie auf den "Fork"-Button oben rechts auf der [GitHub-Seite](https://github.com/nikolaini-byte/blackjack-card-counter).

2. **Repository klonen**
   ```bash
   git clone https://github.com/DEIN-BENUTZERNAME/blackjack-card-counter.git
   cd blackjack-card-counter
   ```

3. **Entwicklungsumgebung einrichten**
   Folgen Sie den Anweisungen in der [Entwicklerdokumentation](Dokumentation-&-Ressourcen#entwicklungsumgebung-einrichten).

4. **Branch erstellen**
   ```bash
   git checkout -b feature/mein-feature
   ```

5. **Änderungen vornehmen und committen**
   ```bash
   git add .
   git commit -m "Beschreibung der Änderungen"
   ```

6. **Änderungen pushen**
   ```bash
   git push origin feature/mein-feature
   ```

7. **Pull Request erstellen**
   Besuchen Sie die [GitHub-Seite](https://github.com/nikolaini-byte/blackjack-card-counter) und klicken Sie auf "Compare & pull request".

## 📋 Beitragsrichtlinien

### Code-Stil

- **JavaScript/TypeScript**: Folgt dem Standard-JavaScript-Stil mit Prettier
- **Python**: PEP 8 mit einer maximalen Zeilenlänge von 88 Zeichen
- **Kommentare**: Dokumentieren Sie komplexe Logik auf Englisch

### Commit-Nachrichten

Verwenden Sie das folgende Format für Commit-Nachrichten:

```
type(scope): kurze Beschreibung

Detaillierte Erklärung der Änderungen, falls notwendig.

Fixes #Issue-Nummer
```

Mögliche Typen:
- `feat`: Neue Funktion
- `fix`: Fehlerbehebung
- `docs`: Dokumentationsänderungen
- `style`: Formatierung, fehlende Semikolons, etc.
- `refactor`: Code-Refactoring
- `test`: Tests hinzufügen oder bestehende ändern
- `chore`: Build-Prozess oder Tooling-Änderungen

### Pull Requests

1. Beschreiben Sie Ihre Änderungen klar und deutlich
2. Verlinken Sie verwandte Issues
3. Stellen Sie sicher, dass alle Tests bestehen
4. Aktualisieren Sie die Dokumentation entsprechend

## 🐛 Fehler melden

Verwenden Sie unsere [Issue-Vorlage](https://github.com/nikolaini-byte/blackjack-card-counter/issues/new?template=bug_report.md), um Fehler zu melden. Bitte geben Sie an:

1. Schritte zum Reproduzieren des Problems
2. Erwartetes Verhalten
3. Tatsächliches Verhalten
4. Screenshots (falls zutreffend)
5. Umgebungsinformationen (Betriebssystem, Browser, Versionen)

## 💡 Feature-Anfragen

Haben Sie eine Idee für ein neues Feature? Erstellen Sie ein [Feature-Request-Issue](https://github.com/nikolaini-byte/blackjack-card-counter/issues/new?template=feature_request.md) mit:

1. Einer klaren Beschreibung des gewünschten Features
2. Der Motivation für dieses Feature
3. Vorschlägen zur Implementierung (optional)
